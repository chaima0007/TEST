# Persona : CTO / Head of Digital / DSI

## Profil type
- **Titre** : Chief Technology Officer / Head of Digital / Directeur des Systèmes d'Information
- **Formation** : Ingénierie informatique, grandes écoles tech (Polytechnique, Centrale, EPFL), MBA tech
- **Ancienneté poste** : 4-10 ans
- **Équipe** : 50-500 personnes (dev, infra, data, cyber, architecture)
- **Reporting** : CEO ou COO

## Motivations profondes
1. **Intégration propre** : API REST bien documentée, pas de "connector" bancal
2. **Sécurité des données** : hébergement EU, RGPD, SOC 2, pas de fuite possible
3. **Scalabilité** : solution qui tient la charge (milliers de fournisseurs, millions d'entrées)
4. **Pas de shadow IT** : outil intégré dans l'écosystème tech validé
5. **Maintenabilité** : documentation, SLA, évolutivité — pas de vendor lock-in opaque

## Douleurs prioritaires
- Solutions ESG = souvent SaaS mal architecturés, peu d'APIs, exports CSV
- RGPD : données fournisseurs (contrats, audits) = données sensibles hébergées où ?
- Intégration SAP/ERP = toujours plus compliquée que promis
- Shadow IT compliance : ESG rachète des solutions sans consulter la DSI
- AI Act : systèmes IA achats/RH utilisés sans classification risque = dette technique

## Ce qui rassure le CTO
- Documentation API OpenAPI/Swagger complète et accessible
- Certifications sécurité (SOC 2 Type II, ISO 27001, hébergement EU AWS/Azure)
- Références intégrations ERP réussies (SAP S/4HANA, Oracle, Workday)
- Modèle de données clair et exportable
- Engagement SLA contractuel (99.9% uptime, temps de réponse API <200ms)

## Ce qui irrite le CTO
- "Notre solution s'intègre facilement à vos systèmes" sans architecture précise
- Pas d'API REST — export/import CSV uniquement
- Données hébergées hors EU sans justification
- Cycle de release lent (<4/an) dans un monde ESG qui évolue vite réglementairement
- Prestataires qui court-circuitent la DSI pour vendre directement à l'ESG

## Argument d'ouverture recommandé
> "Notre API REST est documentée OpenAPI 3.1. Nous hébergeons sur AWS eu-west-1, certifiés SOC 2 Type II. Votre intégration SAP Ariba peut être opérationnelle en 4 semaines. Voulez-vous voir la documentation ?"

## Canal préféré
- Documentation technique publique (GitHub, docs.caelumpartners.eu)
- Email avec lien vers spec technique
- Démo technique (pas commerciale) : sandbox, Postman collection
- Contact via Head of ESG ou CCO qui l'implique en phase technique

## Phrase à éviter absolument
- "C'est simple à intégrer, votre équipe tech n'aura pas besoin de beaucoup de temps."
- "Nos données sont sécurisées." (sans préciser où, comment, quelles certifications)

## Rôle dans le cycle de vente
- Pas décideur business mais **veto décisif** : peut bloquer tout projet mal architecturé
- S'impliquer tôt évite les surprises en phase contractuelle
- Un CTO convaincu = accélération du déploiement × 2
- Proposer un POC technique 4 semaines = approche efficace
